# การปรับปรุงดีไซน์ความคมชัดโหมดมืด-สว่าง และโครงสร้างนำทาง Sidebar

ในการปรับปรุงรอบนี้ ได้โฟกัสเรื่องความถูกต้องตามมาตรฐานการออกแบบเว็บที่ดี (UX/UI Standards) ตามข้อติชมของผู้เรียน โดยอาศัยคำแนะนำจากเครื่องมือช่วยเหลือพัฒนาการออกแบบหน้าจอโลคอล (`frontend-design` skill):

1. **ติดตั้งเครื่องมือช่วยเหลือการออกแบบ (Local Skill)**:
   - ทำการติดตั้ง `anthropics/skills@frontend-design` ลงในระดับโฟลเดอร์โปรเจกต์ (`.agents/skills/frontend-design`) เท่านั้นโดยไม่มีการติดตั้งในแบบ Global เพื่อจำกัดสโคปการใช้คำสั่ง

2. **แก้ปัญหาตัวอักษรกลืนสีโหมดมืด (Dark Mode Contrast)**:
   - ตรวจพบว่าไฟล์เนื้อหาบทเรียนใช้สี Tailwind ที่ไม่มีอยู่จริง (เช่น `dark:text-slate-355`, `dark:text-slate-350`, และ `dark:bg-deepnavy-light`) ทำให้หน้าจอเบราว์เซอร์ไม่แสดงผลสีสีนั้นและกลืนเป็นสีดำมองไม่เห็น
   - ทำการรันสคริปต์แก้ไขเปลี่ยนเป็นสีมาตรฐาน **`dark:text-slate-300`** และ **`dark:bg-slate-900`** ส่งผลให้ข้อความทฤษฎีในโหมดมืดมีความสว่าง คมชัด อ่านง่ายขึ้น 100%

3. **ลดความคอนทราสต์ของบล็อกคำสั่งในโหมดสว่าง (Light Mode Code Block Softening)**:
   - เพิ่ม CSS Override เข้าไปในหน้าเว็บหลัก เพื่อควบคุมว่าในโหมดสว่าง กล่องคำสั่ง `<pre>` จะไม่เป็นสีดำสนิท แต่จะใช้สีพื้นหลังสีเทาอ่อนนุ่มตา (`#f8fafc`) พร้อมขอบเส้นแบ่งสีเบาบาง และใช้ตัวหนังสือสีเข้ม (`#0f172a`) ส่วนคำสั่งในแถว (inline code) จะใช้พื้นหลังสีเทาสว่างคู่กับตัวอักษรสีครามคราม
   - เมื่อผู้ใช้สลับเป็นโหมดมืด ตัวบล็อกคำสั่งจะคืนรูปกลับไปเป็นสีดำครามโปรแสงของระบบคอนโซลอย่างกลมกลืน

4. **ปรับโครงสร้าง Sidebar ให้มีมิติระดับชั้น (Sidebar Hierarchy)**:
   - ขยายขนาดและเพิ่มความหนาตัวอักษรของหัวข้อกลุ่มหลัก (กลุ่ม 1, กลุ่ม 2) เป็น `text-[13px] font-extrabold`
   - ทำการร่นระยะเยื้องหน้า (Indentation) ของบทเรียนหลักแต่ละบทเข้าไปทางขวาด้วยคำสั่ง **`pl-5`**
   - ทำการขยับการเยื้องหน้าของหัวข้อลิงก์ย่อยภายในวิชา (1.1, 1.2...) เข้าไปข้างในลึกขึ้นด้วยระยะเยื้อง **`ml-9 pl-6`** ช่วยให้การแสดงโครงสร้างเมนูมีความลึก ดูเป็นแผนผังย่อยของแต่ละวิชาอย่างเด่นชัด
