# แผนการปรับปรุงระบบค้นหา คอนทราสต์แบบทดสอบ รูปแบบ Grid/Flexbox และการเชื่อมโยงข้อมูล

เอกสารนี้แสดงรายละเอียดแผนการดำเนินการเพื่อพัฒนาและแก้จุดบกพร่องตามที่ผู้เรียนเสนอแนะ โดยครอบคลุมประเด็นสำคัญ 6 ประการ:

---

## 📋 รายละเอียดหัวข้อที่ต้องแก้ไข (Action Items)

### 1. แก้ไขฟอนต์สีเทาในคลาส `space-y-2` (Subtitle) ในโหมดมืด
* **จุดที่เกิดปัญหา:** 
  * หน้าหลัก `index.html` (บรรทัด 231) และส่วนย่อยอื่นๆ มีการใช้คลาสสี `dark:text-slate-350` หรือ `dark:text-slate-355` ซึ่งไม่ใช่ชื่อสีมาตรฐานของ Tailwind CSS ส่งผลให้ระบบเรนเดอร์ข้ามไปเป็นสีดำกลืนกับพื้นหลังโหมดมืด ทำให้อ่านยากมาก
* **แนวทางแก้ไข:** 
  * เปลี่ยนสีฟอนต์ทั้งหมดที่มีปัญหานี้เป็นสีมาตรฐาน **`dark:text-slate-300`** เพื่อให้มีความคมชัด สบายตา และมีโทนสีเดียวกับเนื้อหาหลักส่วนอื่นๆ
  * ตรวจสอบและเปลี่ยนค่าสีนี้ในไฟล์ `index.html`, `lesson/0001-cisco-basics-and-password-reset.html` และ `reference/cisco-switch-quick-reference.html`

### 2. ปรับความคอนทราสต์กล่องแบบทดสอบ (Quiz Box) ในโหมดสว่าง (Light Theme)
* **จุดที่เกิดปัญหา:**
  * ปัจจุบันกล่องคำถามและปุ่มตัวเลือกในแบบทดสอบที่สร้างโดย `assets/app.js` มีการระบุคลาสแบบฮาร์ดโค้ดเป็นโทนสีมืด (`bg-slate-900/60`, `bg-slate-950/60`, `border-slate-800`, `text-slate-100`) ส่งผลให้เมื่อสลับเป็นโหมดสว่าง ตัวกล่องแบบทดสอบยังคงเป็นสีดำสนิท ซึ่งคอนทราสต์เกินไปและไม่สบายตา
* **แนวทางแก้ไข:**
  * ปรับแต่งโค้ดใน `assets/app.js` และ `assets/quiz.css` ให้รองรับธีมแบบสมบูรณ์ (Responsive Toggling):
    * **โหมดสว่าง (Light Mode):** ตัวกล่องเปลี่ยนเป็นพื้นหลังสีขาว (`bg-white` / `var(--card-bg)`) ตัดขอบสีเทาอ่อน (`border-slate-200`) ตัวหนังสือคำถามสีเข้ม (`text-slate-900`) และปุ่มตัวเลือกมีพื้นสีเทาอ่อนนุ่มตาพร้อมเอฟเฟกต์โฮเวอร์เบาๆ
    * **โหมดมืด (Dark Mode):** คืนสไตล์กลับมาเป็นพื้นหลังครามเข้มสวยงามเช่นเดิม
    * ปรับแต่งสีปุ่มที่ตอบถูก/ตอบผิดให้สว่างสบายตาขึ้นเมื่ออยู่ในโหมดสว่าง

### 3. ตรวจสอบการใช้งาน Flexbox และ Grid เพื่อจัดเรียงสัดส่วนหน้าจอทั้งหมด
* **แนวทางการพัฒนา:**
  * ตรวจทานโครงสร้างเลย์เอาต์ (Layout Structuring) ของหน้าเว็บหลักและเพจย่อยทั้งหมด เพื่อให้มั่นใจว่าการจัดหน้าจอใช้ระบบ Flexbox และ Grid ครบถ้วน เพื่อให้การจัดระเบียบสัดส่วนมีความยืดหยุ่น (Responsive) บนอุปกรณ์ขนาดหน้าจอต่างๆ:
    * `index.html` -> ใช้ระบบ Flex แบ่งสัดส่วน Header, Sidebar และ Main area และใช้ Flex ในการจัดวางส่วนตกแต่งย่อย
    * `cisco-switch-quick-reference.html` -> ใช้เลย์เอาต์ Flex คู่กับ Grid ในการแบ่งการ์ดแสดงผลคำสั่ง
    * `0001-cisco-basics-and-password-reset.html` -> ปรับระบบ Layout ให้เป็น Flexbox และ Grid แบบ 2 คอลัมน์ (Sidebar + Main content) เช่นเดียวกับหน้าหลักเพื่อความเป็นหนึ่งเดียวกัน

### 4. เพิ่มปุ่มลิงก์กลับหน้าหลักในเอกสารภายนอก
* **จุดที่จะเพิ่มปุ่ม:**
  * ที่ส่วนหัว (Header) ของหน้า `reference/cisco-switch-quick-reference.html` และหน้า `lesson/0001-cisco-basics-and-password-reset.html` ข้างๆ ปุ่มเปลี่ยนธีม
* **ฟังก์ชันการทำงาน:**
  * ใส่ปุ่มลิงก์ "กลับสู่หน้าหลัก Portal" (`../index.html`) พร้อมไอคอนย้อนกลับ เพื่อให้ผู้เรียนคลิกเปลี่ยนหน้าได้สะดวกโดยไม่ต้องกดปุ่ม Back ของบราว์เซอร์

### 5. พัฒนาระบบค้นหาบทเรียนและหัวข้อย่อยจริง (Interactive Search Engine)
* **จุดที่ปรับเปลี่ยน:**
  * เปลี่ยนกล่องค้นหาใน `index.html` ให้รับค่าอินพุตจริง (`<input type="text" id="search-input">`) พร้อมสร้างกล่องแสดงผลลัพธ์ลอยตัวด้านล่าง (`#search-results`)
* **ฟังก์ชันการทำงาน:**
  * เมื่อผู้เรียนป้อนข้อความค้นหา ระบบ JavaScript ใน `assets/app.js` จะทำการค้นหาคำผ่านข้อมูล `lessonsData` ครอบคลุมทั้งหัวข้อหลัก ชื่อบทเรียน คำอธิบาย และหัวข้อย่อย (เช่น 1.1, 8.2)
  * แสดงผลลัพธ์แมตชิ่งขึ้นในรูปรายการการ์ด พร้อมไฮไลต์คำที่ค้นหา
  * เมื่อกดเลือกรายการ: หน้าเว็บจะสลับเปลี่ยนบทเรียนนั้นๆ ขึ้นมาทำงานทันที และหากเป็นการเลือกหัวข้อย่อย ระบบจะใช้คำสั่ง **Scroll เลื่อนพาผู้ใช้ลงไปยังเป้าหมายไอดีหัวข้อนั้นแบบ Smooth Scrolling** ทันที

### 6. รวบรวมและเพิ่มแหล่งข้อมูลอ้างอิงที่เชื่อถือได้ท้ายแต่ละบทเรียน
* **แนวทางการระบุแหล่งอ้างอิง:**
  * รวบรวมแหล่งศึกษาเพิ่มเติมที่น่าเชื่อถืออย่างเป็นทางการจาก Cisco Systems (เช่น CCNA Study Guide, Whitepapers), RFCs ของ IETF และคู่มือการทำงานด้านเทคนิค
  * เพิ่มฟิลด์โครงสร้าง `references` (มีชื่อเรื่อง, ลิงก์ URL และค่ายืนยันแหล่งที่มา) ในวัตถุบทเรียนแต่ละบทใน `assets/lessons.js`
  * ในหน้าหลักจะแสดงผลแผงสรุปอ้างอิงนี้ที่ตำแหน่งล่างสุดของตัวบทเรียน (ต่อจากกล่องควิซ)
  * ในหน้าเดี่ยว `0001-cisco-basics-and-password-reset.html` จะเพิ่มหมวดหมู่นี้ในตำแหน่งท้ายบทเช่นเดียวกัน
