บทเรียนที่ 1: พื้นฐาน Cisco Switch และการกู้คืนรหัสผ่าน

ยินดีต้อนรับผู้เรียนเข้าสู่บทเรียนแรกของการตั้งค่าอุปกรณ์เครือข่าย Cisco บทเรียนนี้ออกแบบมาเพื่อตอบคำถามของคุณเกี่ยวกับแผนการเรียนรู้ คำสั่งพื้นฐานที่ใช้บ่อย และขั้นตอนการกู้คืนรหัสผ่านที่เป็นมาตรฐาน

💡 สิ่งที่จะได้เรียนรู้ในบทเรียนนี้:
  • แผนการเรียนรู้ระบบเครือข่ายและสวิตช์ Cisco จากพื้นฐาน
  • ตารางคำสั่ง Cisco CLI ที่ใช้บ่อยพร้อมคำอธิบาย
  • วิธีการกู้คืนรหัสผ่าน (Password Recovery) ตามมาตรฐาน Cisco ทั้งแบบเก่าและแบบใหม่

🗺️ 1. เส้นทางการเรียนรู้ Cisco Network (Curriculum Roadmap)

การศึกษา Cisco Network ให้ประสบความสำเร็จควรเรียงลำดับจากง่ายไปยากตามหัวข้อด้านล่างนี้:

  1. พื้นฐาน CLI และโหมดการทำงาน: เข้าใจความแตกต่างระหว่าง User EXEC, Privileged EXEC, Global Configuration และวิธีการสลับโหมด
  2. การตั้งค่าความปลอดภัยเบื้องต้น: การกำหนดรหัสผ่านให้กับ Switch (Enable Secret), การสร้างบัญชีผู้ใช้ในเครื่อง (Local User) และการเข้ารหัสผ่าน
  3. การจัดการหมายเลข IP และการเข้าถึงระยะไกล: การกำหนด IP ในโหมดจัดการ (SVI - VLAN 1), Default Gateway และการเปิดใช้งาน SSH
  4. VLAN และ Trunking: การแบ่งกลุ่มเครือข่ายเสมือน (VLAN), Access Port และการเชื่อมโยงข้อมูลข้ามสวิตช์ด้วย Trunk Port (802.1Q)
  5. การบริหารจัดการไฟล์ระบบ: การบันทึก สำรองข้อมูล (Backup) และกู้คืนไฟล์คอนฟิกูเรชัน (TFTP/FTP)
  6. การกู้คืนระบบและความปลอดภัยขั้นสูง: การทำ Password Recovery และการรีเซ็ตค่าจากโรงงาน (Factory Reset)

🛡️ 2. คำสั่งพื้นฐานที่ใช้บ่อย (Common Basic Commands)

กลุ่มคำสั่งหลักที่คุณจะได้ใช้บ่อยมากที่สุดในการตั้งค่าและตรวจสอบสถานะของสวิตช์:

การสลับโหมดการทำงาน (CLI Navigation)

คำสั่ง คำอธิบาย โหมดเริ่มต้น → โหมดใหม่
enable สลับเข้าสู่โหมด Privileged EXEC (ดูข้อมูลและทดสอบได้เต็มที่) Switch> → Switch#
configure terminal สลับเข้าสู่โหมด Global Configuration (ปรับแต่งแก้ไขค่าต่าง ๆ) Switch# → Switch(config)#
interface [type] [number] เข้าสู่โหมดตั้งค่าการทำงานรายอินเทอร์เฟซ Switch(config)# → Switch(config-if)#
exit ย้อนกลับไปโหมดก่อนหน้าทีละขั้น โหมดใด ๆ → โหมดก่อนหน้า
end (หรือ Ctrl+Z) ย้อนกลับสู่โหมด Privileged EXEC ทันที โหมดแก้ไขใด ๆ → Switch#

การตั้งค่าและตรวจสอบ (Configuration & Verification)

คำสั่ง คำอธิบาย ตัวอย่างการใช้งาน
hostname [name] กำหนดชื่ออุปกรณ์สวิตช์ hostname Dept-Switch-01
enable secret [password] ตั้งรหัสผ่านสำหรับเข้าสู่โหมด Privileged EXEC (เข้ารหัสปลอดภัย) enable secret P@ssw0rd123
show running-config แสดงคอนฟิกูเรชันปัจจุบันที่กำลังทำงานอยู่ใน RAM show running-config
copy running-config startup-config บันทึกคอนฟิกูเรชันปัจจุบันลงใน NVRAM (หรือพิมพ์ write memory) copy running-config startup-config
show ip interface brief แสดงข้อมูลสรุป IP และสถานะพอร์ตเชื่อมโยงแลนทั้งหมด show ip interface brief
show vlan brief แสดงรายชื่อ VLAN และสถานะพอร์ตที่เชื่อมโยงอยู่ show vlan brief

🔐 3. ขั้นตอนการกู้คืนรหัสผ่านสวิตช์ตามมาตรฐาน Cisco

หากคุณลืมรหัสผ่านของสวิตช์ Cisco วิธีการกู้คืนจะขึ้นอยู่กับรุ่นของอุปกรณ์ (Hardware Series) แต่ทุกวิธีจะมีแนวคิดเดียวกันคือ "ต้องข้ามขั้นตอนการอ่าน Startup Configuration ขณะบูตอุปกรณ์"

⚠️ ข้อกำหนดเบื้องต้น:

คุณต้องเชื่อมต่อเครื่องคอมพิวเตอร์เข้ากับสวิตช์ผ่านสาย Console Cable และเข้าถึงทางกายภาพของอุปกรณ์ (Physical Access) เท่านั้น ไม่สามารถดำเนินการผ่าน SSH/Telnet ได้

วิธีที่ A: สำหรับสวิตช์รุ่นเดิม (เช่น Catalyst 2960, 3560, 3750)

  1. ถอดสายไฟสวิตช์ออกเพื่อปิดเครื่อง
  2. กดปุ่ม Mode ค้างไว้ บนแผงด้านหน้าสวิตช์ จากนั้นเสียบสายไฟกลับเข้าไปใหม่เพื่อเปิดเครื่อง
  3. กดปุ่ม Mode ค้างต่อไปจนกระทั่งไฟ LED ของระบบ (System LED) เปลี่ยนเป็นสีเขียวค้าง (หรือกระพริบเร็ว) แล้วจึงปล่อยปุ่ม Mode คุณจะพบกับพรอมต์บูตโหลดเดอร์: 
    switch:
  4. รันคำสั่งเริ่มต้นระบบไฟล์แฟลช: 
    switch: flash_init
  5. ตรวจสอบรายชื่อไฟล์ในหน่วยความจำแฟลช: 
    switch: dir flash:
    (ปกติไฟล์เก็บค่าตั้งค่าจะชื่อ config.text)
  6. เปลี่ยนชื่อไฟล์เพื่อซ่อนค่าคอนฟิกเดิมไม่ให้ระบบนำมาโหลดเปิดเครื่อง: 
    switch: rename flash:config.text flash:config.text.old
  7. สั่งบูตระบบตามปกติ: 
    switch: boot
  8. เมื่อระบบบูตเสร็จ คุณจะอยู่ในโหมดเริ่มต้นที่ยังไม่ได้คอนฟิก ให้ตอบ no ในการตั้งค่าอัตโนมัติ
  9. เข้าสู่โหมด Privileged EXEC (ซึ่งจะไม่มีการถามรหัสผ่าน): 
    Switch> enable
  10. เปลี่ยนชื่อไฟล์คอนฟิกกลับมาเป็นชื่อเดิม: 
    Switch# rename flash:config.text.old flash:config.text
  11. คัดลอกไฟล์คอนฟิกเดิมกลับเข้าสู่หน่วยความจำที่กำลังใช้งาน: 
    Switch# copy flash:config.text system:running-config
    (ระวัง! ห้ามสั่ง write memory ในขั้นตอนนี้เด็ดขาด เพราะจะเซฟค่าว่างทับค่าเดิม)
  12. เข้าโหมดคอนฟิกและตั้งรหัสผ่านใหม่: 
    Switch# configure terminal
Switch(config)# enable secret [รหัสผ่านใหม่]
Switch(config)# exit
  13. บันทึกการเปลี่ยนแปลง: 
    Switch# copy running-config startup-config

วิธีที่ B: สำหรับสวิตช์รุ่นใหม่ (เช่น Catalyst 9000 Series บน IOS-XE)

  1. เชื่อมต่อสาย Console เปิดโปรแกรม Terminal จากนั้นปิดและเปิดสวิตช์ใหม่
  2. กดปุ่ม Ctrl+C บนคีย์บอร์ดซ้ำ ๆ ระหว่างบูตช่วงแรกเพื่อเข้าโหมด ROMMON: 
    switch:
  3. ตั้งค่าข้ามการโหลดไฟล์ Startup Config: 
    switch: SWITCH_IGNORE_STARTUP_CFG=1
  4. สั่งบูตระบบปฏิบัติการหลัก: 
    switch: boot
  5. เมื่อบูตเสร็จ สวิตช์จะไม่มีคอนฟิกเก่าทำงาน ให้เข้าโหมดผู้ใช้ระดับสูง: 
    Switch> enable
  6. โหลดค่าคอนฟิกเก่ากลับมาทำงานใน RAM: 
    Switch# copy startup-config running-config
  7. เข้าไปกำหนดสิทธิ์และรหัสลับใหม่: 
    Switch# configure terminal
Switch(config)# username admin privilege 15 secret [รหัสผ่านใหม่]
  8. (สำคัญมาก) ต้องสั่งยกเลิกคำสั่งข้าม Startup Config เพื่อให้การเปิดเครื่องครั้งหน้าบูตตามปกติ: 
    Switch(config)# no system ignore startupconfig switch all
  9. บันทึกและเสร็จสิ้นขั้นตอน: 
    Switch(config)# exit
Switch# copy running-config startup-config
