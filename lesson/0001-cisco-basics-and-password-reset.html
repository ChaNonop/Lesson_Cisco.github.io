บทเรียนที่ 1: พื้นฐาน Cisco Switch และการกู้คืนรหัสผ่าน

ยินดีต้อนรับผู้เรียนเข้าสู่บทเรียนแรกของการตั้งค่าอุปกรณ์เครือข่าย Cisco บทเรียนนี้ออกแบบมาเพื่อตอบคำถามของคุณเกี่ยวกับแผนการเรียนรู้ คำสั่งพื้นฐานที่ใช้บ่อย และขั้นตอนการกู้คืนรหัสผ่านที่เป็นมาตรฐาน

💡 สิ่งที่จะได้เรียนรู้ในบทเรียนนี้
  • แผนการเรียนรู้ระบบเครือข่ายและสวิตช์ Cisco จากพื้นฐาน
  • ตารางคำสั่ง Cisco CLI ที่ใช้บ่อยพร้อมคำอธิบาย
  • วิธีการกู้คืนรหัสผ่าน (Password Recovery) ตามมาตรฐาน Cisco ทั้งแบบเก่าและแบบใหม่

1. เส้นทางการเรียนรู้ Cisco Network (Curriculum Roadmap)

การศึกษา Cisco Network ให้ประสบความสำเร็จควรเรียงลำดับจากง่ายไปยากตามหัวข้อด้านล่างนี้:

  1. พื้นฐาน CLI และโหมดการทำงาน: เข้าใจความแตกต่างระหว่าง User EXEC, Privileged EXEC, Global Configuration และวิธีการสลับโหมด
  2. การตั้งค่าความปลอดภัยเบื้องต้น: การกำหนดรหัสผ่านให้กับ Switch (Enable Secret), การสร้างบัญชีผู้ใช้ในเครื่อง (Local User) และการเข้ารหัสผ่าน
  3. การจัดการหมายเลข IP และการเข้าถึงระยะไกล: การกำหนด IP ในโหมดจัดการ (SVI - VLAN 1), Default Gateway และการเปิดใช้งาน SSH
  4. VLAN และ Trunking: การแบ่งกลุ่มเครือข่ายเสมือน (VLAN), Access Port และการเชื่องโยงข้อมูลข้ามสวิตช์ด้วย Trunk Port (802.1Q)
  5. การบริหารจัดการไฟล์ระบบ: การบันทึก สำรองข้อมูล (Backup) และกู้คืนไฟล์คอนฟิกูเรชัน (TFTP/FTP)
  6. การกู้คืนระบบและความปลอดภัยขั้นสูง: การทำ Password Recovery และการรีเซ็ตค่าจากโรงงาน (Factory Reset)

2. คำสั่งพื้นฐานที่ใช้บ่อย (Common Basic Commands)

ด้านล่างนี้คือกลุ่มคำสั่งหลักที่คุณจะได้ใช้บ่อยมากที่สุดในการตั้งค่าและตรวจสอบสถานะของสวิตช์:

การสลับโหมดการทำงาน (CLI Navigation)

คำสั่ง คำอธิบาย โหมดเริ่มต้น → โหมดใหม่
enable สลับเข้าสู่โหมด Privileged EXEC (ดูข้อมูลและทดสอบได้เต็มที่) Switch> → Switch#
configure terminal สลับเข้าสู่โหมด Global Configuration (ปรับแต่งแก้ไขค่าต่าง ๆ) Switch# → Switch(config)#
interface [type] [number] เข้าสู่โหมดตั้งค่าการทำงานรายอินเทอร์เฟซ (เช่น interface gigabitethernet 1/0/1) Switch(config)# → Switch(config-if)#
exit ย้อนกลับไปโหมดก่อนหน้าทีละขั้น โหมดใด ๆ → โหมดก่อนหน้า
end (หรือ Ctrl+Z) ย้อนกลับสู่โหมด Privileged EXEC ทันที โหมดการแก้ไขใด ๆ → Switch#

การตั้งค่าและตรวจสอบ (Configuration & Verification)

คำสั่ง คำอธิบาย ตัวอย่างการใช้งาน
hostname [name] กำหนดชื่ออุปกรณ์สวิตช์ hostname Dept-Switch-01
enable secret [password] ตั้งรหัสผ่านที่ถูกเข้ารหัสเพื่อเข้าสู่โหมด Privileged EXEC (แนะนำใช้ตัวนี้) enable secret P@ssw0rd123
show running-config แสดงคอนฟิกูเรชันปัจจุบันที่กำลังทำงานอยู่ใน RAM (ค่าจะหายไปหากไม่ได้เซฟและไฟดับ) show running-config
copy running-config startup-config บันทึกคอนฟิกูเรชันปัจจุบันลงใน NVRAM (ค่าจะยังคงอยู่แม้ปิดเครื่อง) หรือใช้ย่อว่า write memory copy running-config startup-config
show ip interface brief แสดงสถานะอินเทอร์เฟซ (พอร์ต) ทั้งหมดแบบย่อ รวมถึงหมายเลข IP Address show ip interface brief
show vlan brief แสดงรายชื่อ VLAN และสถานะพอร์ตที่เชื่อมโยงกับ VLAN นั้น ๆ show vlan brief

3. ขั้นตอนการกู้คืนรหัสผ่านสวิตช์ตามมาตรฐาน Cisco

หากคุณลืมรหัสผ่านของสวิตช์ Cisco วิธีการกู้คืนจะขึ้นอยู่กับรุ่นของอุปกรณ์ (Hardware Series) แต่ทุกวิธีจะมีแนวคิดเดียวกันคือ "ต้องข้ามขั้นตอนการอ่าน Startup Configuration ขณะบูตอุปกรณ์"

⚠️ ข้อกำหนดเบื้องต้น

คุณต้องเชื่อมต่อเครื่องคอมพิวเตอร์เข้ากับสวิตช์ผ่านสาย Console Cable และเข้าถึงทางกายภาพของอุปกรณ์ (Physical Access) เท่านั้น ไม่สามารถดำเนินการผ่าน SSH/Telnet ได้

วิธีที่ A: สำหรับสวิตช์รุ่นเดิม (เช่น Catalyst 2960, 3560, 3750)

  1. ถอดสายไฟสวิตช์ออกเพื่อปิดเครื่อง
  2. กดปุ่ม Mode ค้างไว้ บนแผงด้านหน้าสวิตช์ จากนั้นเสียบสายไฟกลับเข้าไปใหม่เพื่อเปิดเครื่อง
  3. กดปุ่ม Mode ค้างต่อไปจนกระทั่งไฟ LED ของระบบ (System LED) เปลี่ยนเป็นสีเขียวค้าง (หรือสั่นกระพริบเร็วในบางรุ่น) แล้วจึงปล่อยปุ่ม Mode คุณจะพบกับพรอมต์บูตโหลดเดอร์: 
    switch:
  4. รันคำสั่งเริ่มต้นระบบไฟล์แฟลช: 
    switch: flash_init
  5. (ระวัง) ตรวจดูรายชื่อไฟล์เพื่อหาไฟล์คอนฟิกูเรชัน: 
    switch: dir flash:
    (ปกติไฟล์ค่าตั้งค่าจะชื่อ config.text)
  6. เปลี่ยนชื่อไฟล์เพื่อซ่อนค่าคอนฟิกเดิมไม่ให้ระบบนำมาเปิดตอนบูต: 
    switch: rename flash:config.text flash:config.text.old
  7. สั่งเริ่มการทำงานระบบ: 
    switch: boot
  8. เมื่อระบบบูตเสร็จ คุณจะอยู่ในโหมดเริ่มต้นที่ยังไม่ได้คอนฟิก ให้ตอบ no ในกล่องคำถามเพื่อข้ามการตั้งค่าแบบอัตโนมัติ
  9. เข้าสู่โหมด Privileged EXEC (จะข้ามการถามรหัสผ่านเนื่องจากไม่มีค่าคอนฟิก): 
    Switch> enable
  10. เปลี่ยนชื่อไฟล์คอนฟิกกลับมาเป็นชื่อเดิม: 
    Switch# rename flash:config.text.old flash:config.text
  11. คัดลอกไฟล์คอนฟิกกลับเข้าสู่หน่วยความจำที่กำลังใช้งาน: 
    Switch# copy flash:config.text system:running-config
    (ระวัง! อย่าใช้ copy running startup ในขั้นตอนนี้เด็ดขาด เพราะจะเซฟค่าคอนฟิกว่างเปล่าทับค่าเดิม)
  12. เข้าโหมดคอนฟิกและตั้งรหัสผ่านใหม่: 
    Switch# configure terminal
Switch(config)# enable secret [รหัสผ่านใหม่ของคุณ]
Switch(config)# exit
  13. บันทึกการเปลี่ยนแปลง: 
    Switch# copy running-config startup-config

วิธีที่ B: สำหรับสวิตช์รุ่นใหม่ (เช่น Catalyst 9000 Series บนระบบปฏิบัติการ IOS-XE)

  1. เชื่อมต่อสาย Console เปิดโปรแกรม Terminal
  2. ปิดสวิตช์และเปิดสวิตช์ใหม่
  3. กดปุ่ม Ctrl+C บนคีย์บอร์ดซ้ำ ๆ (หรือกดปุ่ม Mode ค้างไว้ที่ตัวอุปกรณ์) ระหว่างการบูตช่วงแรก จนกระทั่งเข้าสู่โหมด ROMMON: 
    switch:
  4. ตั้งค่าตัวแปรสิ่งแวดล้อมเพื่อข้ามการโหลด Startup Configuration: 
    switch: SWITCH_IGNORE_STARTUP_CFG=1
  5. พิมพ์ boot เพื่อเริ่มบูตสวิตช์: 
    switch: boot
  6. เมื่อบูตเสร็จ สวิตช์จะไม่มีคอนฟิกเก่าทำงาน ให้เข้าโหมดผู้ใช้ระดับสูง: 
    Switch> enable
  7. กู้คืนไฟล์คอนฟิกเดิมกลับมารัน: 
    Switch# copy startup-config running-config
  8. ตั้งค่ารหัสผ่านใหม่: 
    Switch# configure terminal
Switch(config)# username admin privilege 15 secret [รหัสผ่านใหม่ของคุณ]
  9. (สำคัญมาก) ต้องสั่งยกเลิกคำสั่งข้าม Startup Config เพื่อให้รอบหน้าเครื่องบูตตามปกติ: 
    Switch(config)# no system ignore startupconfig switch all
    (สำหรับบางรุ่น หากคำสั่งข้างต้นไม่รองรับ ให้ใช้คำสั่ง no system ignore startup-config หรือย้อนกลับไปเคลียร์ตัวแปรใน ROMMON)
  10. บันทึกการเปลี่ยนแปลงและรีสตาร์ตเพื่อทดสอบ: 
    Switch(config)# exit
Switch# copy running-config startup-config

4. แหล่งข้อมูลอ้างอิงอย่างเป็นทางการ (Reliable References)

สำหรับข้อมูลเชิงลึกและอัปเดตอย่างเป็นทางการ คุณสามารถศึกษาเพิ่มเติมได้จากลิงก์ด้านล่างนี้:

  • คู่มือ Password Recovery สำหรับสวิตช์ Catalyst 9000 - จากเว็ปไซต์อย่างเป็นทางการของ Cisco
  • คู่มือ Password Recovery สำหรับสวิตช์ตระกูล Catalyst 2960, 3560, 3750 - จาก Cisco
  • Cisco Official Website - ค้นหาคู่มือเพิ่มเติมสำหรับสวิตช์ตระกูลอื่น ๆ
