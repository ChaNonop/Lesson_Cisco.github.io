  - รองรับการสร้าง VLAN และ Trunk (`vlan [ID]`, `switchport mode access/trunk`, `switchport access vlan [ID]`)

## 📚 3. สรุปเนื้อหาบทเรียนที่พร้อมใช้งาน (Lessons 1 - 8)

### บทเรียนที่ 1: สถาปัตยกรรมแบบ Layer และ OSI Model L1-L2
- **1.1 สถาปัตยกรรมแบบ Layer:** แนวคิดแบ่งหน้าที่เป็นขั้น เพื่อความยืดหยุ่นในการพัฒนาและแก้ปัญหาเครือข่าย
- **1.2 Physical Layer (L1):** รับส่งสัญญาณดิบในรูปไฟฟ้า แสง หรือคลื่นวิทยุ (หน่วยข้อมูลคือ Bits)
- **1.3 Data Link Layer (L2):** จัดกลุ่มบิตข้อมูลเป็นเฟรม (Frame) และระบุที่อยู่ทางกายภาพ (MAC Address)
- **1.4 การทำงานของ Switch:** บันทึกที่อยู่พอร์ตลง MAC Address Table และสลับสัญญาณแบบจุดต่อจุด ป้องกันการสับสนของข้อมูล

### บทเรียนที่ 2: เจาะลึก L3 & L4 - IP Routing และ TCP vs UDP
- **2.1 Network Layer (L3):** นำส่งข้อมูลข้ามเครือข่ายด้วยเลข IP Address และเลือกเส้นทางโดย Router (หน่วยข้อมูลคือ Packet)
- **2.2 Transport Layer (L4):** จัดสรรพอร์ตการให้บริการและบริหารช่องทางการสื่อสาร (หน่วยข้อมูลคือ Segment)
  - **TCP:** แข็งแกร่ง เชื่อถือได้ (Connection-oriented) มีระบบตอบรับแพ็กเก็ต
  - **UDP:** เน้นความเร็วสูงแบบส่งข้อมูลแล้วไม่รอสถานะรับ (Connectionless) เหมาะสำหรับสตรีมมิ่ง/DNS
- **2.3 TCP 3-Way Handshake:** ขั้นตอนเริ่มต้นสื่อสารอย่างปลอดภัย 3 จังหวะ (SYN ➡️ SYN-ACK ➡️ ACK)

### บทเรียนที่ 3: ระดับชั้นบนของ OSI L5-L7 และโปรโตคอลสำคัญ
- **3.1 กลุ่มเลเยอร์ระดับบน:** ทำงานใกล้ชิดกับแอปพลิเคชันและการแสดงข้อมูลสำหรับผู้ใช้
- **3.2 Session Layer (L5):** ควบคุมเซสชันการล็อกอินและทิศทางการสื่อสาร ( NetBIOS, SQL )
- **3.3 Presentation Layer (L6):** แปลงโครงสร้างรูปแบบไฟล์ บีบอัดข้อมูล และเข้ารหัสลับความปลอดภัย (SSL/TLS)
- **3.4 Application Layer (L7):** เชื่อมโยงซอฟต์แวร์ เช่น HTTP/HTTPS, DNS (แปลงชื่อเว็บเป็น IP), DHCP (แจกจ่าย IP แอดเดรสอัตโนมัติ)

### บทเรียนที่ 4: การเดินทางและการแปลงร่างของข้อมูล (Encapsulation Process)
- **4.1 ขั้นตอนส่ง (Encapsulation):** ห่อหุ้มข้อมูลจากบนลงล่าง: Data ➡️ Segment (TCP) ➡️ Packet (IP) ➡️ Frame (MAC + FCS) ➡️ Bits
- **4.2 ขั้นตอนรับ (Decapsulation):** ปลายทางแกะหัวควบคุมของซองออกย้อนกลับขึ้นบนตามลำดับเพื่อเข้าถึงเนื้อความจริง

### บทเรียนที่ 5: ก้าวแรกสู่ Cisco IOS CLI และการใช้โหมดต่าง ๆ
- **5.1 การเข้าถึงสวิตช์ (Console):** การเชื่อมต่อสายผ่านพอร์ตและเปิดโปรแกรมจำลองหน้าจอด้วยค่า 9600 Baud
- **5.2 โหมดต่าง ๆ ของ Cisco IOS:**
  - `User EXEC (>)` - สิทธิ์จำกัดดูสถานะทั่วไป
  - `Privileged EXEC (#)` - สิทธิ์ตรวจสอบเชิงลึก (พิมพ์สลับเข้าโดย `enable`)
  - `Global Config ((config)#)` - สิทธิ์แก้ไขปรับแต่งหลัก (พิมพ์สลับเข้าโดย `configure terminal`)
  - `Sub-modes` - โหมดตั้งค่าพอร์ตย่อยและล็อกอิน (`(config-if)#` หรือ `(config-line)#`)
- **5.3 คีย์ลัดช่วยป้อนคำสั่ง:** วิธีใช้เครื่องหมาย `?`, ปุ่ม **Tab** ช่วยเติมคำ และย้อนกลับด้วย `exit`/`end`

### บทเรียนที่ 6: การตั้งค่าพารามิเตอร์พื้นฐานและความปลอดภัยสวิตช์
- **6.1 การจัดการ RAM vs NVRAM:** ความแตกต่างของ Running Config (RAM หายเมื่อไฟดับ) และ Startup Config (NVRAM ถาวร บันทึกค่าด้วยคำสั่ง `write memory` หรือ `copy run start`)
- **6.2 คำสั่งตั้งค่าพื้นฐาน:**
  - ตั้งชื่อสวิตช์: `hostname [ชื่อ]`
  - รหัสความปลอดภัย: `enable secret [พาสเวิร์ด]`
  - ป้องกันดูข้อมูลเปล่า: `service password-encryption`
  - ข้อความแบนเนอร์แจ้งเตือนล็อกอิน: `banner motd #[ข้อความ]#`

### บทเรียนที่ 7: การตั้งค่า IP Management และความปลอดภัยระยะไกล SSH
- **7.1 SVI & Management IP:** การตั้งค่าไอพีบนพอร์ตเสมือน Interface VLAN 1 เพื่อใช้ควบคุมสวิตช์จากทางไกล
- **7.2 Default Gateway บน Switch:** การใส่เกตเวย์ให้สวิตช์ส่งข้อมูลย้อนกลับข้ามวงแลนไปหาผู้ดูแลระบบได้
- **7.3 ช่องโหว่ของโปรโตคอล Telnet:** ชี้แจงถึงอันตรายของการส่งผ่านรหัสผ่าน Plain Text ในระบบ
- **7.4 ขั้นตอนติดตั้ง SSH v2:** ขั้นตอนละเอียดในการสร้างโดเมนเนม บัญชีแอดมินระดับ 15, รัน RSA 2048 บิต และบังคับให้ล็อกอินเข้าเฉพาะ SSH เท่านั้น

### บทเรียนที่ 8: การจัดการเครือข่ายด้วย VLAN และการเชื่อมต่อ Trunk
- **8.1 VLAN คืออะไร?:** การแยก Broadcast Domain ขจัดสแปมทราฟฟิก และเพิ่มระดับความปลอดภัยภายใน
- **8.2 Access Port vs Trunk Port:** พอร์ต Access สำหรับเครื่องลูกข่าย (1 VLAN) และพอร์ต Trunk สลับสวิตช์ (ส่งหลาย VLAN)
- **8.3 มาตรฐาน IEEE 802.1Q:** โครงสร้างการฝังข้อมูลแท็ก VLAN ID ลงในเฟรมแลนบนสาย Trunk
- **8.4 คอนฟิก VLAN & Trunk:** คำสั่งสร้าง VLAN, ตั้งชื่อแผนก, บังคับพอร์ตสังกัด VLAN, เปิดลิงก์ Trunk และคำสั่งตรวจสอบความถูกต้อง

## 🛠️ 5. สรุปคำสั่งพื้นฐานยอดนิยม (Cisco CLI Cheat Sheet)

| คำสั่ง | โหมดที่ใช้ | คำอธิบายการทำงาน |
| :--- | :---: | :--- |
| `enable` | `Switch>` | สลับเข้าสู่โหมดสิทธิ์แอดมิน |
| `configure terminal` | `Switch#` | สลับเข้าสู่โหมดแก้ไขตั้งค่าหลัก |
| `hostname [ชื่อ]` | `Switch(config)#` | กำหนดชื่อสวิตช์ |
| `enable secret [รหัสผ่าน]` | `Switch(config)#` | ตั้งรหัสผ่านเข้าโหมดสิทธิ์แอดมินแบบเข้ารหัสลับแน่นหนา |
| `service password-encryption` | `Switch(config)#` | เข้ารหัสผ่านที่เป็น plaintext ทั้งระบบ |
| `banner motd #[ข้อความ]#` | `Switch(config)#` | ตั้งข้อความเตือนภัย MOTD ก่อนล็อกอิน |
| `write memory` (หรือ `wr`) | `Switch#` | บันทึกการตั้งค่าลง NVRAM (เซฟข้อมูล) |
| `copy running-config startup-config` | `Switch#` | บันทึกการตั้งค่าลง NVRAM (มาตรฐานหลัก) |
| `show running-config` | `Switch#` | แสดงค่าคอนฟิกูเรชันที่รันอยู่ใน RAM |
| `show ip interface brief` | `Switch#` | แสดงตารางข้อมูล IP และสถานะเปิด/ปิดของพอร์ตทั้งหมด |
| `show vlan brief` | `Switch#` | แสดงตารางรายชื่อ VLAN และพอร์ตที่จัดสรรอยู่ |
| `show interfaces trunk` | `Switch#` | ตรวจสอบพอร์ตเชื่อมต่อหลัก (Trunk) และ VLAN ที่ได้รับอนุญาต |
| `exit` | ทุกโหมด | ย้อนกลับโหมดก่อนหน้า 1 ขั้น |
| `end` (หรือ Ctrl+Z) | โหมดแก้ไขย่อย | ดีดตัวเองกลับสู่โหมด Privileged EXEC (`#`) ทันที |

## 🔐 6. สรุปขั้นตอนการกู้คืนรหัสผ่าน (Password Recovery Standard)

### สำหรับรุ่นเดิม (Catalyst 2960 / 3560 / 3750)
1. ดึงสายไฟออกเพื่อปิดเครื่อง
2. **กดปุ่ม Mode ค้างไว้** บนตัวสวิตช์แล้วเสียบสายไฟกลับ
3. ปล่อยปุ่ม Mode เมื่อไฟ System LED สีเขียวค้าง (จะเข้าสู่หน้าจอ `switch:`)
4. พิมพ์ `flash_init` เพื่อเริ่มต้นใช้งานแฟลชไดรฟ์สวิตช์
5. พิมพ์คำสั่งเปลี่ยนชื่อไฟล์คอนฟิกเดิมเพื่อหลบการโหลดตอนเปิดเครื่อง:
   `rename flash:config.text flash:config.text.old`
6. บูตระบบด้วยคำสั่ง `boot`
7. บูตเสร็จพิมพ์คำสั่ง `enable` เพื่อข้ามการขอรหัสผ่าน
8. เปลี่ยนชื่อไฟล์กลับเป็นชื่อเดิม:
   `rename flash:config.text.old flash:config.text`
9. โหลดไฟล์เดิมเข้ามาเพื่อความปลอดภัยของข้อมูลเก่า:
   `copy flash:config.text system:running-config`
10. เข้าไปแก้ไขรหัสผ่านใหม่:
    `configure terminal` ➡️ `enable secret [รหัสผ่านใหม่]` ➡️ `exit`
11. บันทึกข้อมูลและใช้งานตามปกติ: `copy running-config startup-config`

### สำหรับรุ่นใหม่ (Catalyst 9000 / IOS-XE)
1. เชื่อมต่อสายคอนโซล แล้วปิด-เปิดสวิตช์ใหม่
2. กดปุ่ม **Ctrl+C** (หรือปุ่ม Mode ค้าง) บนคีย์บอร์ดซ้ำ ๆ ระหว่างบูต เพื่อเข้าสู่โหมด ROMMON (`switch:`)
3. สั่งระบบให้ ignore ข้ามคอนฟิกเดิม:
   `SWITCH_IGNORE_STARTUP_CFG=1`
4. พิมพ์คำสั่ง `boot`
5. เมื่อระบบบูตเสร็จสิ้นพิมพ์ `enable`
6. กู้คืนค่าคอนฟิกเก่ากลับมา:
   `copy startup-config running-config`
7. เข้าไปเปลี่ยนรหัสผ่านแอดมินใหม่:
   `configure terminal` ➡️ `username admin privilege 15 secret [รหัสผ่านใหม่]`
8. **(สำคัญมาก)** ปิดคำสั่งให้ข้ามคอนฟิกเพื่อให้รอบหน้าบูตปกติดังเดิม:
   `no system ignore startupconfig switch all`
9. ออกและบันทึกข้อมูล:
   `exit` ➡️ `copy running-config startup-config`

## 📋 7. รายงานผลการรีวิวและปรับปรุงโค้ด (Code Quality & Technical Accuracy Review)

ได้รับการตรวจสอบความถูกต้องของระบบบทเรียนและไฟล์สคริปต์ทั้งหมดผ่านกระบวนการ **Parallel Code Review** โดยมีรายละเอียดประเด็นที่ได้รับการแก้ไข ดังนี้:

- **การแก้ไขด้านมาตรฐานโค้ด (Standards):**
  - ล้างคลาสสี Tailwind ผิดรูปแบบทั้งหมดกว่า 15 จุด (เช่น `-650`, `-850` และ `-250` ซึ่งไม่มีอยู่ในระบบ Tailwind CSS)
  - เพิ่ม **typeof Guard** และ **Dispatch Map** ใน [app.js](file:///D:/My_Doc_lesson/บทเรียน%20Cisco/assets/app.js) เพื่อเพิ่มเสถียรภาพและป้องกันปัญหาโค้ดหยุดทำงานหากไฟล์เสริมล้มเหลว
  - อัปเกรด HTML ให้สนับสนุนการอ่านออกเสียงโดยเครื่องมือช่วยเหลือผู้พิการ (Accessibility) ผ่านการเพิ่ม ARIA Attributes
  - แก้ไขคลาส toggling sidebar เมนูแอ็กทิฟให้ตรงกับลักษณะจริง
- **การแก้ไขความถูกต้องทางวิชาการ (Technical Accuracy):**
  - ปรับข้อมูลช่วงไอดี VLAN ใน [lessons.js](file:///D:/My_Doc_lesson/บทเรียน%20Cisco/assets/lessons.js) ให้ถูกต้องครอบคลุม Extended-range (1006–4094)
  - แก้ไขข้อคำอธิบาย PDU ใน Transport Layer ให้แสดงข้อมูลทั้ง Segment (TCP) และ Datagram (UDP)
  - ปรับคำแปลศัพท์เทคนิค (MD5 hash แบบทางเดียว แทนคำว่า คลื่นแฮช)
  - สะกดคำถูกต้องสมบูรณ์ (เช่น โดเนม ➡️ โดเมน) และขจัดข้อความซ้ำ
  - อัปเกรดลิงก์อ้างอิงท้ายบทเรียน (RFC 2616 ➡️ RFC 9110 ล่าสุด, ลิงก์ Cisco Document ที่เฉพาะเจาะจงมากขึ้น)
