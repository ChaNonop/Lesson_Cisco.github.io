🛡️ 1. การตั้งค่าความปลอดภัยและระบบขั้นต้น

ใช้กำหนดพารามิเตอร์พื้นฐานและรหัสผ่านเมื่อนำอุปกรณ์ขึ้นติดตั้ง

เปลี่ยนชื่อเครื่องและเปิดใช้งานการเข้ารหัสลับรหัสผ่าน: 
Switch# configure terminal
Switch(config)# hostname SW-Primary
SW-Primary(config)# enable secret P@ssword1234
SW-Primary(config)# service password-encryption
กำหนดรหัสผ่านให้กับพอร์ต Console (เมื่อต่อสายตรง): 
SW-Primary(config)# line console 0
SW-Primary(config-line)# password ConsolePass1
SW-Primary(config-line)# login
SW-Primary(config-line)# exit

🌐 2. การจัดการ Remote IP & เปิดใช้งาน SSH v2

ใช้เพื่อควบคุมและจัดการสวิตช์ระยะไกลแทนการต่อสายตรงหน้าเครื่อง

ตั้งหมายเลข IP ในพอร์ตเสมือน (SVI) และ Default Gateway: 
SW-Primary(config)# interface vlan 1
SW-Primary(config-if)# ip address 192.168.1.10 255.255.255.0
SW-Primary(config-if)# no shutdown
SW-Primary(config-if)# exit
SW-Primary(config)# ip default-gateway 192.168.1.254
สร้างผู้ใช้โลคอลและเปิดบริการ SSH v2.0 (ปิด Telnet ที่ไม่ปลอดภัย): 
SW-Primary(config)# ip domain-name companynet.com
SW-Primary(config)# username admin privilege 15 secret AdminPass99
SW-Primary(config)# crypto key generate rsa  (เลือก Modulus = 2048)
SW-Primary(config)# ip ssh version 2
SW-Primary(config)# line vty 0 15
SW-Primary(config-line)# transport input ssh
SW-Primary(config-line)# login local
SW-Primary(config-line)# exit

🔌 3. การกำหนดค่า VLAN และการทำ Trunking (802.1Q)

ใช้เพื่อจัดโครงสร้างแผนกในองค์กรและเชื่อมต่อทราฟฟิกระหว่างอุปกรณ์

สร้าง VLAN และเอาพอร์ตแลนเข้าไปสังกัด (Access Mode): 
SW-Primary(config)# vlan 10
SW-Primary(config-vlan)# name Sales
SW-Primary(config-vlan)# exit
SW-Primary(config)# interface fastethernet 0/5
SW-Primary(config-if)# switchport mode access
SW-Primary(config-if)# switchport access vlan 10
SW-Primary(config-if)# exit
ตั้งค่าพอร์ตให้รับส่งข้อมูลทุก VLAN เพื่อต่อหาสวิตช์อีกตัว (Trunk Mode): 
SW-Primary(config)# interface gigabitethernet 1/0/1
SW-Primary(config-if)# switchport mode trunk
SW-Primary(config-if)# switchport trunk allowed vlan 10,20
SW-Primary(config-if)# exit

💾 4. การบันทึก สำรองข้อมูล และการล้างเครื่องค่าโรงงาน

ใช้บันทึกการทำงาน ตรวจสอบสถานะการเชื่อมต่อ และการลบระบบ

คำสั่งบันทึกการทำงานและล้างคอนฟิก (Factory Reset): 
SW-Primary# copy running-config startup-config  (บันทึกข้อมูล)
SW-Primary# write erase                        (ล้างข้อมูลตั้งค่า)
SW-Primary# delete flash:vlan.dat              (ลบฐานข้อมูล VLAN)
SW-Primary# reload                             (เริ่มต้นอุปกรณ์ใหม่)
คำสั่งใช้ตรวจสอบสถานะการทำงานยอดนิยม: 
SW-Primary# show ip interface brief    (ตรวจดูพอร์ตและ IP แอดเดรส)
SW-Primary# show interface status      (ตรวจดูสถานะการอัพดาวน์และความเร็วพอร์ต)
SW-Primary# show vlan brief            (ตรวจเช็กคู่สาย VLAN ที่กำหนดค่า)
SW-Primary# show mac address-table     (ดูตารางเชื่อมโยงพอร์ตและ MAC ของลูกข่าย)

🔑 5. ขั้นตอนกู้คืนรหัสผ่านสวิตช์ (Password Recovery)

สำหรับดำเนินการเมื่อลืมรหัสผ่านผู้ควบคุมระบบ

ซีรีส์เดิม (Catalyst 2960/3560/3750)

  1. ถอดสายไฟปิดเครื่อง
  2. กดปุ่ม **Mode** ค้างไว้แล้วเสียบปลั๊กไฟ
  3. ปล่อยปุ่ม Mode เมื่อไฟ SYST สีเขียวค้าง
  4. ป้อนคำสั่ง: flash_init
  5. เปลี่ยนชื่อไฟล์เพื่อซ่อนค่าคอนฟิก: 
    rename flash:config.text flash:config.text.old
  6. ป้อนคำสั่ง: boot
  7. ระบบบูตเสร็จป้อน enable
  8. ดึงไฟล์คืน: 
    rename flash:config.text.old flash:config.text
copy flash:config.text system:running-config
  9. เข้า Global Config และตั้งรหัสผ่านใหม่: 
    enable secret [รหัสผ่านใหม่]
  10. เซฟทับด้วย write memory

ซีรีส์ใหม่ (Catalyst 9000 - IOS-XE)

  1. ต่อสาย Console ปิดและเปิดสวิตช์ใหม่
  2. กดปุ่ม **Ctrl+C** ค้างซ้ำ ๆ จนเข้าสู่หน้าจอ switch:
  3. สั่งให้ระบบข้ามคอนฟิกเดิม: 
    SWITCH_IGNORE_STARTUP_CFG=1
  4. ป้อนคำสั่ง: boot
  5. บูตเสร็จเข้าโหมดแอดมิน: enable
  6. ดึงค่าคอนฟิกเก่าคืน: 
    copy startup-config running-config
  7. เข้าไปกำหนดชื่อผู้ใช้และรหัสผ่านใหม่ในโหมด config
  8. **(สำคัญ)** สั่งยกเลิกตัวแปร Ignore บูต: 
    no system ignore startupconfig switch all
  9. บันทึกและตรวจสอบ: write memory
